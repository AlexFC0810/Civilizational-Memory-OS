# Creator Leverage Graph

**Status:** Working framework  
**Purpose:** Define the visual and interactive layer for routing high-fidelity knowledge to the creators, audiences, and moments where it can generate the greatest marginal impact.

## Core thesis

A directory answers:

> Who exists?

A leverage graph answers:

> **Who can move which audience on which issue, with what degree of trust, using which ready asset, at what expected marginal impact and distortion risk?**

The graph is not a leaderboard of fame. It is a decision-support system for intelligent allocation of research, production, outreach, and creator support.

## Node types

- creators;
- audiences and communities;
- claim families;
- narrative strains;
- topics;
- source cards;
- experts and researchers;
- public assets;
- recurring media formats;
- institutions and action pathways.

## Edge types

- creator-to-audience trust;
- creator-to-topic credibility;
- creator-to-format strength;
- audience overlap;
- claim-to-source dependency;
- claim-to-creator fit;
- creator-to-expert relationship;
- asset-to-audience performance;
- narrative-to-institution implication;
- feedback-to-canonical revision.

Every edge should carry provenance, date, confidence, and update conditions where practical.

## Primary visual views

### 1. Pareto frontier

Plot creators or interventions by expected truth impact versus cost and distortion risk.

The frontier reveals options where no alternative delivers more impact at equal or lower cost and risk.

### 2. Audience coverage map

Show which communities are already served, oversaturated, neglected, or reachable only through particular trusted messengers.

### 3. Narrative outbreak map

Show active claim families by prevalence, velocity, harm, audience salience, and evidentiary tractability.

### 4. Asset readiness map

For each active claim family, show whether the system has:

- verified verdict;
- source card;
- short response;
- long explainer;
- visual asset;
- fair-context clip;
- translated version;
- creator-ready packet;
- correction update.

### 5. Creator opportunity map

Show the best next packet for each creator and the best next creator for each claim family.

### 6. Network fragility map

Reveal single points of failure, excessive dependence on one creator or platform, audience silos, and missing expert links.

## Opportunity score

A provisional creator-topic opportunity score can combine:

- trusted reach;
- audience relevance;
- topic credibility;
- timeliness;
- creator interest;
- adoption probability;
- expected reuse;
- transfer potential;
- cross-network spillover;
- asset readiness;
- correction reliability;
- distortion and capture risk;
- coordination cost.

The score should remain inspectable. It must not become an opaque ranking system pretending to objective precision.

## Interactive behavior

A user should be able to:

1. select a claim family;
2. see the accepted verdict and evidence state;
3. identify audiences currently encountering it;
4. see creators with the strongest trust and fit;
5. inspect available assets;
6. generate a Creator Supercharge Packet;
7. record outreach and creator feedback;
8. track publication and audience response;
9. feed objections and corrections into the AIOS;
10. view the next recommended action.

## Pareto modes

### Maximum immediate reach

Prioritize large trusted distribution when the claim is urgent and source-closed.

### Maximum transfer

Prioritize creators and formats most likely to teach a reusable reasoning skill.

### Maximum neglected coverage

Prioritize communities that receive little high-quality information.

### Maximum trust repair

Prioritize creators able to correct misinformation inside their own ideological or religious coalition.

### Maximum institution conversion

Prioritize audiences and partners capable of turning the insight into policy, curriculum, funding, or a real institution.

The system should make the selected objective explicit.

## Ethical constraints

- no covert psychological targeting;
- no scraping private audience data;
- no political microtargeting based on sensitive personal traits;
- no pretending probability scores are moral judgments about people;
- no creator manipulation through concealed incentives;
- no centralized script enforcement;
- no optimizing outrage merely because it performs;
- no routing a claim before the evidence state supports it.

## MVP visual

The first MVP can be a GitHub-backed interactive graph with four entity types:

- ten creators;
- ten claim families;
- ten source-hardened assets;
- five audience clusters.

The MVP only needs to answer:

1. What is the highest-leverage creator-topic pairing now?
2. What asset is missing before outreach?
3. What audience is currently uncovered?
4. What did the last creator deployment teach the AIOS?

## Canonical lines

> **Do not route truth by follower count alone. Route it by trust, fit, timing, transfer, and marginal impact.**

> **A leverage map turns distribution from guesswork into a corrigible civilizational strategy.**
