# Creator Truth Desk v1.3 — 80/20 Acceptance Gate

> **⛓ BINDING HEADER (added 2026-07-20 by the CoS at merge; body unedited).** This gates **product decisions, not claims** — a different domain from `scripts/evals.mjs`, so the two never compete:
> - **This file:** "should we build/accept this change to the Truth Desk?" Acceptance criteria for an interface.
> - **`scripts/evals.mjs`:** "may this historical claim be said in public?" Evidence adjudication.
> A change may pass every gate here and still contain nothing publishable; a gated claim needs nothing from this file to ship. No claim becomes deploy-facing by satisfying these criteria.
>
> **Software status unchanged:** the Creator Truth Desk itself remains **PARKED** behind demonstrated creator value (`MASTER_PLAN.md` row 17 — no software before manual demand-proof). This file is the acceptance standard for *when* it is built, not authorization to build it now.

**Status:** Proposal eval gate v0.1  
**Purpose:** Prevent Claude Code from implementing attractive changes that weaken source precision, live usability, or truth constraints.

---

# Gate 1 — Source Constitution

A first-time user can explain, without prompting, the difference between:

- Qur’an;
- Prophetic Sunnah;
- hadith/sira report;
- tafsir/theology/fiqh;
- institution/state;
- movement/culture;
- individual conduct;
- historical fruit.

Fail if the interface implies:

- Qur’an-only doctrine;
- hadith and Sunnah are identical;
- every canonical hadith is certain history;
- every Muslim state is Islam;
- history is a source of revelation.

---

# Gate 2 — Five-Second Live Retrieval

At 390 × 844 CSS pixels, a user can retrieve each benchmark answer in under five seconds:

1. “Convert, pay tax, or die.”
2. “Jihad means holy war.”
3. “Sharia will replace the Constitution.”
4. “Muhammad married a nine-year-old.”
5. “Women are worth half.”
6. “Muslims are taking over America.”

Each Live result shows only:

- seven-second line;
- burden question;
- true concession;
- reversal.

Fail if evidence expansion is required to locate the core answer.

---

# Gate 3 — Hostile Screenshot

Extract every high-emphasis line without surrounding context.

Fail if any isolated line plausibly communicates:

- “truth is only internal; public messaging is spin”;
- “all hadith are unreliable”;
- “Muslims should ignore Sunnah”;
- “another demographic is the real enemy”;
- “documented abuse should be denied”;
- “Golden Age success proves every Islamic doctrine”;
- “the Church deliberately translated the Bible into Latin solely to suppress reading”;
- factual certainty beyond the source state.

---

# Gate 4 — Aisha Source Precision

The card must correctly distinguish:

- no Qur’anic age;
- dominant later traditional report;
- multiple extant versions/chains;
- Little’s common-archetype argument;
- no proven exact alternative age;
- separate claims of prepubescence, coercion, injury, and psychiatric diagnosis.

Fail on any of:

- “one chain”;
- “Little proved eighteen/nineteen”;
- “Little proved all hadith unreliable”;
- denial that the canonical report exists;
- contextualization before challenging evidence and inference.

---

# Gate 5 — Same Standard

Apply the actor/doctrine/institution distinction to:

- a Muslim criminal;
- a corrupt priest;
- a Christian state;
- a Muslim state;
- a secular political movement.

Pass only if the model:

- refuses collective guilt;
- does not erase institutional responsibility;
- requires the same source and inference standard for all groups.

---

# Gate 6 — Fruit Test Calibration

A universal smear can be tested against historical counterexamples.

Pass only if the interface explicitly distinguishes:

- what the counterexample disproves;
- what it does not prove;
- which institution produced the outcome;
- whether the source is closed.

Fail if Golden Age history is treated as flawless, sacred, or sufficient by itself.

---

# Gate 7 — Opponent Steelman

Run four adversarial reviews:

## Sunni traditionalist

Can the reviewer fairly accuse the desk of Qur’an-only framing, erasing Sunnah, or caricaturing hadith sciences?

## Secular historian

Can the reviewer find anachronism, false certainty, source laundering, or use of Golden Age mythology as proof?

## Christian critic

Can the reviewer show that the priest analogy is applied asymmetrically or that Christian texts are treated by a lower standard?

## Hostile media editor

Can the reviewer create a damaging but fair clip from a self-certifying, demographic, conspiratorial, or dishonest-sounding line?

All high-severity findings block outreach.

---

# Gate 8 — Privacy and Accessibility

Pass only if:

- third-party font/network claims are accurate;
- feedback storage is explained exactly;
- astrolabe nodes are keyboard operable;
- focus states are visible;
- reduced motion is supported;
- text alternatives exist;
- mobile has no horizontal overflow;
- copy targets meet practical touch size.

---

# Gate 9 — Provenance

Every externally deployable factual claim includes:

- source reference;
- confidence state;
- prohibited overclaim;
- correction mechanism;
- research owner or queue if incomplete.

Fail if a design badge such as `source-closed` is not backed by an inspectable source packet.

---

# Gate 10 — Baseline Superiority

The v1.3 proposal replaces v1.2 only if it is demonstrably better on:

- source-layer comprehension;
- live retrieval speed;
- factual precision;
- screenshot safety;
- mobile usability;
- accessibility;
- user confidence;

A more sophisticated page that is slower or harder to use fails.

> **Complexity must justify itself at the point of use.**
