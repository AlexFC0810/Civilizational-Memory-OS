# EXP-001 — Broad Reconnaissance vs Micro Source Closure

**Status:** Active research-method experiment  
**Created:** 2026-07-13  
**Program:** Prophetic Civilizational Science  
**Related issues:** #50, #51, #52  
**Operating protocol:** `protocols/FOUNDER_ARCHITECT_CLAUDE_RESEARCH_ORGANISM.md`

---

## 1. Experiment question

> For a new civilizational research domain, does a broad source-universe reconnaissance prompt or a narrow source-closure packet produce more trustworthy and reusable progress per unit of effort?

This experiment does not test which historical conclusion is preferred. It tests which research architecture more reliably earns conclusions.

---

## 2. Background

### Variant A already executed

The first Byzantium Deep Research run attempted to map:

- major periods;
- primary corpora;
- manuscripts and editions;
- institutions;
- moral architecture;
- adherence gaps;
- Islamic and Song control cases;
- future infrastructure.

It generated useful leads but failed the source-first gate because it:

- relied heavily on tertiary discovery sources;
- did not source-close a primary passage;
- mixed inspected and uninspected sources;
- expanded scope before verifying foundations;
- discussed source independence without demonstrating it;
- created apparent completeness without completed provenance chains.

This is the baseline condition.

### Variant B to execute

Claude Code will source-close one bounded object:

**The Pantokrator Typikon hospital evidence.**

It must complete at least one chain:

`claim -> passage -> original Greek -> critical edition -> manuscript basis -> published translation -> context -> implementation evidence or explicit absence -> counterevidence -> confidence -> safe wording`

---

## 3. Pre-registered prediction

Variant B is expected to:

- produce fewer total claims;
- produce a much higher source-closure rate;
- reduce unsupported specificity;
- make source dependence visible;
- generate more reusable schemas and validation requirements;
- reveal more uncertainty;
- produce at least one bounded claim suitable for later integration.

Variant A may remain superior for:

- discovering unknown source clusters;
- identifying archives and specialists;
- generating an initial terrain map;
- detecting possible comparison cases.

The likely result is not that one variant replaces the other, but that the correct sequence is:

> **brief reconnaissance -> ranked acquisition queue -> micro source closure -> calibrated expansion**

---

## 4. Variants

### Variant A — Broad reconnaissance

**Input:** Byzantine source-universe prompt.  
**Unit:** Civilization-scale terrain.  
**Primary goal:** Discover source clusters and research branches.  
**Failure risk:** Breadth creates false evidentiary confidence.

### Variant B — Micro source closure

**Input:** Pantokrator Typikon source-closure packet.  
**Unit:** One document and one institutional claim cluster.  
**Primary goal:** Complete passage-to-public-wording provenance.  
**Failure risk:** Local depth may not generalize and may miss broader context.

---

## 5. Shared constraints

Both variants must:

- distinguish primary, specialist, synthesis, and discovery-only sources;
- record access status;
- search for counterevidence;
- avoid final theological verdicts;
- preserve uncertainty;
- produce GitHub-ready artifacts;
- prohibit self-canonization;
- expose source dependence;
- separate norm, prescription, practice, and causal inference.

## 7. Acceptance condition for Variant B

Variant B passes only if:

1. at least one important claim has an exact source locator;
2. original Greek and edition metadata are preserved;
3. manuscript basis or uncertainty is visible;
4. a published translation is identified and compared;
5. prescription is separated from implementation;
6. source dependence is shown;
7. limiting or breaker evidence is included;
8. a confidence judgment explains its reasons;
9. safe public wording is materially narrower than prohibited overstatement;
10. another agent can reconstruct the conclusion without relying on fluent summary.

---

## 8. Decision rule

After Variant B completes, the architect will classify the result as one of:

- **B wins:** micro source closure should become the default first execution unit after minimal reconnaissance;
- **A wins:** broad mapping created more source-closed progress than expected;
- **Sequence wins:** minimal A followed by B is superior to either alone;
- **No conclusion:** inputs, effort, or evaluation were not comparable enough.

No result may be selected because it produces the preferred Byzantine conclusion.

---

## 9. Expected protocol changes

If the pre-registered prediction holds:

- broad reconnaissance will receive a strict stopping rule;
- every source-universe run must end with a ranked micro-excavation queue;
- no broad report may enter canon directly;
- the first execution packet must close one primary-source chain;
- expansion occurs only after the pipeline passes on a bounded object;
- Claude Code commands should distinguish `/recon-map` from `/source-close`;
- evaluator scripts should measure closure rather than citation volume.
