# The Worthy Goal Protocol — how a Chief of Staff sets a goal worth pursuing

**Written 2026-07-20** at the operator's request, after the goal below was set and met in a single session. Portfolio-general: nothing here is specific to this repo. **Candidate for promotion to `doctrine-hub`** once it has proven itself in a second repo (per the hub's own rule: promote patterns only after they prove useful in real work).

## The goal that prompted this

> **"One human steers a truth-factory that outpaces distortion."**
> The mission needs ~50 load-bearing facts gated; we have 9. The bottleneck isn't intelligence — it's **coordination physics**. This session makes Civilizational-Memory-OS the portfolio's first fully coordination-compliant node: seats that close themselves, receipts that survive session death, and doctrine that cannot rot unread — enforced by code, not by attention.

It was met the same session: archive 9 → 13, four seats harvested, a silent id-collision caught and made impossible, two protocols written. What made it work was not ambition. It was **structure** — and the structure is reusable.

## The five tests of a worthy goal

A goal is worth a session only if it passes all five. Most goals that feel exciting fail #3 or #5.

**1. It names the real bottleneck, not the visible symptom.**
The visible symptom was "the seats look paused." The real bottleneck was that nobody could tell what four parallel seats had produced. Ask: *if I solved the thing I'm about to work on, would the system actually move — or would the same jam reappear next week under a new name?*

**2. It is one sentence a tired human can hold.**
"One human steers a truth-factory that outpaces distortion." If the goal needs a paragraph, it is a plan wearing a goal's clothes. The sentence is what survives context loss, compaction, and the founder's attention moving elsewhere.

**3. It has a receipt — something that is true after, and demonstrably false before.**
Not "improve coordination." Rather: *the gate exits 1 on a duplicate id, verified by deliberately reintroducing the collision.* If you cannot state what artifact will exist, you have set a mood, not a goal.

**4. It compounds — the next cycle starts ahead of this one.**
An immune system in the gate makes every future parallel cycle safer, forever, at no recurring cost. A goal that only produces output ("card five more claims") is work; a goal that produces **capacity** is leverage. Prefer the goal that changes what is possible next time.

**5. It survives the founder losing interest.**
Anything whose value depends on the operator staying engaged is a hobby, not infrastructure. Encode the win as a file, a check, or a law — never as a habit someone must remember.

## How to find one (the ten-minute method)

1. **Read reality first, not the request.** Sync the branches, list the sessions, check the queue. The gap between what the operator *believes* is happening and what *is* happening is where the worthy goal usually hides. (This session: four seats believed paused had in fact all delivered, and their sessions were gone.)
2. **Name the bottleneck out loud, and check it's a bottleneck and not a preference.** A bottleneck has evidence: a queue that isn't draining, a number that isn't moving, a failure that already cost something.
3. **Ask what only this seat can see.** The Chief of Staff sits across lanes; it sees collisions no individual seat can. That cross-lane view is the goal's natural raw material — a single seat cannot set this goal, which is exactly why it's worth the CoS's session.
4. **Write the sentence. Then write the receipt.** If the receipt won't exist by end of session, shrink the goal until it will. A met small goal beats an admired large one.
5. **State it to the operator before working.** Not to ask permission — to let them redirect cheaply while redirection is still cheap.

## The failure modes this kills

The goal that is really a to-do list · the goal that produces admiration instead of artifacts · the goal that solves the loudest complaint rather than the deepest jam · the goal that requires the founder's continued attention to stay true · the goal so large that nothing lands, so the session ends with a plan instead of a receipt · and the subtlest one: **the goal that optimizes activity rather than verified world-state change** (Nexus Dispatch's law).

## Standing rule for this seat

Every session that receives *"you have the wheel"* opens by setting a goal against these five tests and stating it plainly — one sentence, one receipt — before the first tool call that changes anything.
