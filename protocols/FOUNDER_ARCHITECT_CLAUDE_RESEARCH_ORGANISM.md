## 1. Core decision

ChatGPT should not be the default deep-research execution engine for this project.

The highest-leverage division of labor is:

- **Founder:** plants high-signal seeds, detects moral stakes, anomalies, false frames, and opportunities.
- **ChatGPT / AI architect:** converts seeds into maps, hypotheses, research architecture, experiments, evaluation criteria, integrations, and strategic decisions.
- **Claude Code:** performs repository-scale research execution, source acquisition, corpus inspection, coding, validation, and durable implementation.
- **Independent agents and specialists:** challenge, replicate, and falsify.
- **GitHub:** stores the memory, state, provenance, experiments, corrections, and current truth status.
- **Reality:** judges the system.

Compressed:

> **The founder senses. The architect frames. Claude executes. The red team attacks. GitHub remembers. Reality decides.**

---

## 2. Why this allocation is superior

The project loses leverage when the architect becomes consumed by manual retrieval, citation formatting, and long execution loops.

The architect’s scarce value is:

- identifying the question beneath the question;
- connecting distant domains;
- detecting hidden assumptions and category errors;
- choosing the correct unit of analysis;
- constructing causal models and comparison classes;
- identifying the highest-information experiment;
- designing falsifiers and breaker-evidence requirements;
- integrating contradictory results;
- preserving the golden thread across many projects;
- deciding what should exist next.

Claude Code is better positioned for:

- sustained repository work;
- large corpus acquisition and processing;
- repeated source lookups;
- structured extraction;
- schema implementation;
- validation and test automation;
- source and claim graphs;
- parallel agents;
- reproducible pipelines;
- converting protocols into working machinery.

This is not a hierarchy of intelligence. It is an allocation of cognition by comparative advantage.

## 3. The seven-stage organism

### Stage 1 — Signal

**Owner:** Founder

Inputs may include:

- a video;
- screenshot;
- historical claim;
- moral outrage;
- intuition;
- contradiction;
- analogy;
- question nobody seems to ask;
- opportunity to help a movement or field.

The founder does not need citations or a perfect formulation.

Required output:

- seed;
- why it matters;
- suspected opportunity;
- emotional or moral signal;
- any initial constraints.

### Stage 2 — Architecture

**Owner:** ChatGPT / AI architect

The architect determines:

- the real question;
- whether the signal is novel, duplicated, or connected to existing canon;
- the unit of analysis;
- source layers;
- historical and causal comparison classes;
- likely confounders;
- hypotheses;
- falsifiers;
- artifact routing;
- research sequence;
- execution agent;
- success criteria;
- stopping rule.

Required output:

- opportunity map;
- exact research packet;
- ranked next moves;
- evaluation rubric;
- integration target.

### Stage 3 — Execution

**Owner:** Claude Code

Claude Code:

- reads the governing repository context;
- acquires and inspects sources;
- executes bounded research packets;
- works from primary texts and critical editions where available;
- creates translation cards and source records;
- builds datasets, graphs, scripts, and tests;
- records inaccessible sources and uncertainty;
- searches for breaker evidence;
- commits durable artifacts;
- opens or updates issues and pull requests.

Claude Code may propose conclusions. It may not canonize its own work.

### Stage 4 — Independent challenge

**Owner:** Separate red-team agent, model, or specialist

The challenger should not share the scout’s desired conclusion.

It tests:

- source authenticity and edition choice;
- mistranslation;
- source dependence;
- selective quotation;
- comparison asymmetry;
- causal overreach;
- omitted victims or control cases;
- apologetic escape hatches;
- hostile framing tricks;
- confidence inflation.

### Stage 5 — Integration

**Owner:** ChatGPT / AI architect

The architect reconciles:

- scout findings;
- breaker evidence;
- source quality;
- unresolved disputes;
- comparison results;
- project-wide implications.

It then proposes:

- accepted claims;
- provisional verdicts;
- rejected claims;
- revised frameworks;
- new research dependencies;
- public-safe language;
- what the result changes elsewhere in the system.

### Stage 6 — Deployment

**Owners:** Architect + Claude Code + founder approval where material

Claude Code packages and publishes artifacts. The architect ensures the force plane does not outrun the truth plane.

Potential outputs:

- canonical documents;
- public essays;
- datasets;
- diagrams;
- videos;
- websites;
- prompts;
- educational tools;
- coordination infrastructure;
- policy or movement resources.

### Stage 7 — Reality feedback

**Owner:** Whole organism

Collect:

- expert criticism;
- user misunderstanding;
- failed predictions;
- broken workflows;
- missing evidence;
- public response;
- adoption;
- downstream usefulness;
- unintended harm;
- contradictions with later findings.

Every meaningful failure must produce a protocol, prompt, schema, eval, routing, or canon update.

## 4. Default routing rules

### Route to ChatGPT / architect when the task is primarily:

- discovering what should exist;
- reframing the problem;
- identifying unknown unknowns;
- connecting projects and disciplines;
- building the ontology;
- choosing comparisons;
- designing packets and experiments;
- integrating many prior findings;
- deciding what is highest leverage;
- creating public compression from accepted evidence;
- auditing whether the mission is drifting.

### Route to Claude Code when the task is primarily:

- deep source research;
- reading many files or large corpora;
- original-language source collection;
- repository implementation;
- schema and tool building;
- claim extraction at scale;
- citation checking;
- datasets and timelines;
- code, tests, automation, and visualization;
- updating many connected artifacts;
- running repeatable research workflows.

### Route to an independent model or specialist when:

- a conclusion is load-bearing;
- manuscript or philological expertise matters;
- theological commitments create bias risk;
- a causal inference is unusually consequential;
- expert disagreement remains live;
- the scout and integrator may share the same blind spot.

### ChatGPT may inspect primary sources directly when:

- adjudicating a disputed passage;
- spot-checking Claude Code’s claims;
- identifying a translation or reasoning anomaly;
- performing a small high-information test;
- integrating evidence across several languages or fields;
- the direct source is necessary to decide the next architecture move.

This is selective judicial review, not default manual execution.

## 7. Split-test doctrine

We do not split-test truth.

We split-test **methods for reaching, preserving, and transmitting truth**.

Valid experiment targets:

- research-packet size;
- prompt structure;
- source hierarchy;
- agent role separation;
- scout model;
- red-team model;
- sequential versus parallel research;
- original-language-first versus translation-first workflow;
- claim-ledger schemas;
- confidence representation;
- human-review timing;
- public framing;
- content format;
- retrieval and memory methods;
- evaluation rubrics.

Invalid optimization targets:

- which answer receives more engagement;
- which theology is easier to defend;
- which claim flatters the founder or audience;
- which version hides uncertainty most effectively;
- which conclusion wins the argument regardless of evidence.

## 9. Evaluation dimensions

### Truth quality

- source-closure rate;
- direct-primary-source ratio;
- independent-witness ratio;
- correction rate after red team;
- unsupported-claim rate;
- translation integrity;
- causal calibration;
- uncertainty honesty.

### Research efficiency

- time to source-closed claim;
- cost per resolved uncertainty;
- number of orphaned artifacts;
- duplicated work;
- scope-drift incidents;
- percentage of output reused downstream.

### Systems quality

- reproducibility;
- provenance completeness;
- schema compliance;
- broken-link rate;
- integration latency;
- ability of a new agent to reconstruct the reasoning.

### Strategic quality

- importance of uncertainty resolved;
- downstream leverage;
- number of projects unlocked;
- decision relevance;
- moral stakes;
- tractability;
- optionality created.

### Force quality

- comprehension by cold audiences;
- memorability;
- resistance to misinterpretation;
- ability to survive expert scrutiny;
- usefulness for education, coordination, or action.

A method may improve speed while damaging truth. The evaluation must preserve the tradeoff rather than producing one vanity score.

## 10. Recursive self-improvement loop

`Signal -> architecture -> experiment -> execution -> red team -> integration -> deployment -> feedback -> failure classification -> protocol patch -> next experiment`

Classify failures as:

- question failure;
- scope failure;
- source failure;
- translation failure;
- causal failure;
- comparison failure;
- agent-routing failure;
- memory failure;
- integration failure;
- force failure;
- execution failure;
- mission failure.

For every consequential failure:

1. preserve valid work;
2. identify the failed assumption;
3. write the correction;
4. modify the relevant artifact;
5. add or update an eval;
6. test the replacement on a known case;
7. propagate the improvement to related projects.

> **An error that only produces an apology is wasted. An error that upgrades the operating system becomes capital.**

---

## 11. Anti-bottleneck and anti-drift rules

1. The founder does not become the manual task manager for every agent.
2. ChatGPT does not default to doing Claude Code’s sustained execution work.
3. Claude Code does not silently redefine the mission or canonize its own findings.
4. The same agent should not research, red-team, integrate, and approve a load-bearing conclusion without independent review.
5. More agents do not automatically mean better research.
6. Split tests must have shared inputs and explicit evaluation criteria.
7. A new framework is not created when an existing one can be extended.
8. Research infrastructure must serve real questions, not become a self-justifying software project.
9. Public output is not the finish line; reality feedback must return to the system.
10. The mission outranks the elegance of the machinery.

---

## 12. The architect’s recurring questions

At every major junction, ask:

- What is the hidden question?
- What are we assuming without noticing?
- Which uncertainty is load-bearing?
- What is the smallest experiment that could change the map?
- Which source or comparison could falsify our preferred view?
- What can Claude Code execute that should not consume architect attention?
- What requires architect judgment rather than more retrieval?
- What should be connected to another project?
- What should exist that does not yet?
- Are we building knowledge, machinery, content, coordination, or an institution?
- What would make this useful after the founder and current models are gone?

- ChatGPT owns opportunity detection, architecture, packet design, causal models, integration, and strategic synthesis.
- Claude Code owns the Pantokrator source-closure research, original-source acquisition, claim extraction, repository machinery, and implementation.
- An independent red-team pass must attack the resulting dossier.
- ChatGPT integrates the scout and red-team outputs into a versioned verdict.
- The calibration triangle begins only after the source-closure pipeline proves itself.

The Deep Research prompts already created remain useful as specifications and test fixtures, but ChatGPT does not need to execute them itself.
